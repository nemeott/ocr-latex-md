\begin{abstract}
	Optical Character Recognition has seen many advancements recently, including improved accuracy via Large Language Models (LLMs),
	and enhanced layout understanding through computer vision techniques like structured segmentation. While these improvements have improved
	accuracy and complex format parsing, handwritten content still remains a challenge. Current methods for converting documents to
	Markdown and LaTeX mainly focus on digitally created or printed text, with little attention given to handwritten material.
	To combat this issue, we propose a multi-model approach, consisting of one Convolutional Neural Network encoder and two
	Recurrent Neural Network decoders: LaTeX and Markdown. First we segment the image, identifying text and math blocks.
	Then we choose the appropriate model according to the classification criteria; if the confidence does not meet a specific threshold,
	we use both models and select the best output. Further analyses are in progress. We expect this project to contribute to the improvement of
	handwritten content extraction.

\section{Introduction}

The problem we aim to solve with this project is converting handwritten text and math to Markdown and LaTeX.
Currently, many models can convert handwritten math into LaTeX, but the problem here is that many people write their calculations on paper.
The latest models do not usually support handwritten content or are largely inaccurate for such writing. This is where our underlying problem lies.
We aim to create a hybrid system that can deal with handwritten math and text and convert it into Markdown and LaTeX.
Primary users of our system would be students and instructors, or others who frequently write notes and calculations by hand.
The majority of students use paper for proofs and calculations, and converting them to LaTeX can be challenging, especially for new users.
Conversion can be time-consuming, as LaTeX code for equations can be quite long and complex. This is where our math-text hybrid conversion
approach would come into use, converting handwritten notes. Our solution removes the manual conversion time and skill challenge.
We propose a hybrid solution that classifies text and math, and then uses corresponding decoder models to extract the content.

\section{Related Works}

The task of converting handwritten mathematics into LaTeX has a few notable recent attempts. Both a Support Vector Machine (SVM) classifier and
Convolution Neural Network (CNN) showed success in converting individual characters to LaTeX \citep{chang2016painfree}. For handwriting samples with
multiple characters, SVM can also lead to misclassification in times where handwriting is highly diverse, such as the multiple ways of writing
the character `x,' in which a CNN performs significantly better \citep{schechter2017converting}.
Other testing indicates that encoder-decoder paradigms show promising results as they vastly improve on the ``counting problem,''
where complicated fractions or integrals lead to omitted or hallucinated characters as a result of the inherent non-linearity of mathematical expressions \citep{wang2020recognizinghandwrittenmathematicalexpressions}.

As transformers have become the dominant encoder-decoder technique, analysis comparing them to CNN-RNN
(Convolutional Neural Network--Recurrent Neural Network) methods shows potentially even better results \citep{sundararaj2024automatedlatexcodegeneration}.

While previous attempts use the single-encoder single-decoder paradigm, we intend to take it a step further by using two decoders.
This will allow us to do a full conversion of handwriting to Markdown and LaTeX as opposed to previous attempts solely converting to LaTeX.
We also noticed that ensemble modeling has been successful for optical character recognition \citep{preiss2025evaluationensemblelearningtechniques}.
Combining this with the diversity problem of handwritten mathematical expressions, we intend to explore techniques to boost model diversity in
ensemble approaches such as clustering as a preprocessing step for ensemble modeling, which has been successful in other
classification applications \citep{cui2021386}. Specifically, clustering the dataset and performing SVM on each resultant cluster,
only requiring each model to be responsible for a significantly smaller portion of the dataset overall.

For our implementation, we will use a classifier to determine which output is more appropriate, with a confidence metric for each option.
If neither option reaches the minimum confidence metric, then both will be produced and the best output will be selected.
For handwritten math to LaTeX we use the MathWritingHandwritten dataset \citep{yo2025mathwritinghandwritten},
which has over 250k rows of handwritten mathematical expressions with the corresponding LaTeX code.
For parsing regular handwritten text we use the IAM Handwriting Database \citep{marti2002iam, teklia2024handwrittentext},
which has a large dataset of labeled handwritten text.

In our implementation we would use an Optical Character Recognition model, which uses a RNN and techniques such as
segmentation and contrast manipulation to isolate text. Then a binary classifier with a confidence metric could be used to determine
whether to produce LaTeX or Markdown. If neither option reaches the required threshold, both would be produced using the
respective decoders, and the best outcome would be used.
